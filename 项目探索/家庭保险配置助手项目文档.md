## 1. 项目概述

### 1.1 项目定位

**"小白家庭的保险顾问"** —— 不推销，只帮你算清楚该买什么、买多少、是否重复。

### 1.2 核心价值主张

| 维度 | 说明 |
|------|------|
| **目标用户** | 有家庭保障需求但不懂保险的普通用户（25-45岁） |
| **核心痛点** | 不知道买什么、不知道买多少、不知道是否重复 |
| **解决方案** | 保障缺口分析 + 中立产品对比 + 保单管理 |
| **差异化** | 中立不推销、小白友好、全市场产品对比 |

### 1.3 产品形态

- **平台**：微信小程序（降低使用门槛）
- **功能**：保障缺口分析、产品对比、保单管理、理赔指南
- **定位**：工具型产品，不做保险销售

## 2. 市场分析

### 2.1 市场规模

| 维度 | 数据 |
|------|------|
| 中国家庭数量 | 约4.5亿户 |
| 有保险需求的家庭 | 约1.5亿户（30%） |
| 目前有完整保障的家庭 | ＜10% |
| 保险配置工具潜在用户 | 约1-1.5亿人 |

### 2.2 竞争格局

| 竞品类型 | 代表产品 | 优势 | 劣势 |
|----------|----------|------|------|
| 保险销售平台 | 水滴保、小雨伞 | 产品丰富、价格透明 | 推销导向、不中立 |
| 代理人工具 | 保险师、多保鱼 | 功能专业 | 不适合小白用户 |
| 综合金融平台 | 支付宝保险、微信保险 | 用户量大 | 产品杂乱、无规划功能 |

**我们的机会**：专注"家庭保障规划"，中立不推销，小白友好。

### 2.3 目标用户画像

#### 用户1：新手爸爸/妈妈（28-35岁）
- **特征**：刚有孩子，风险意识增强，但不懂保险
- **痛点**：想给孩子买保险，但不知道买什么、买多少
- **付费意愿**：强（为了孩子愿意花钱）

#### 用户2：职场白领（25-35岁）
- **特征**：有一定收入，开始考虑家庭保障
- **痛点**：网上信息太多太杂，不知道听谁的
- **付费意愿**：中等（需要看到价值）

#### 用户3：有房一族（30-45岁）
- **特征**：有房有贷，家庭责任重
- **痛点**：担心自己出事，家庭陷入困境
- **付费意愿**：强（风险意识强）

## 3. 产品设计

### 3.1 核心功能模块

#### 模块1：保障缺口分析（核心功能）

**功能描述**：
用户输入家庭信息→系统自动计算保障缺口→输出保障建议

**输入项**：
- 家庭成员（年龄、性别、职业）
- 收入情况（月收入、年收入）
- 负债情况（房贷、车贷）
- 已有保单（可选，拍照上传或手动录入）

**输出项**：
- 保障缺口报告（重疾/医疗/寿险/意外各差多少）
- 优先级建议（先买什么、后买什么）
- 预算分配建议（每月/每年需要多少保费）

**计算规则**（简化版）：
```
重疾险缺口 = 家庭年收入 × 3 - 已有重疾保额
医疗险缺口 = 100万 - 已有医疗保额
寿险缺口 = 家庭负债 + 3年收入 - 已有寿险保额
意外险缺口 = 50万 - 已有意外保额
```

---

#### 模块2：保险产品对比

**功能描述**：
用户选择2-3款产品→系统横向对比→输出对比报告

**对比维度**：
- 价格（保费/年）
- 保额
- 保障范围
- 免责条款
- 理赔率（如有公开数据）
- 用户评价

**数据来源**：
- 公开产品信息（保险公司官网）
- 用户众测数据
- 合作平台提供的数据

---

#### 模块3：保单管理

**功能描述**：
用户上传保单→系统自动识别→统一管理→到期提醒

**功能点**：
- 保单拍照上传（OCR识别）
- 保单手动录入
- 保单分类管理（重疾/医疗/寿险/意外）
- 到期/缴费提醒
- 保单分享给家人

**OCR识别字段**：
- 保险公司
- 产品名称
- 被保人
- 保额
- 保费
- 缴费期限
- 到期日

---

#### 模块4：理赔指南

**功能描述**：
用户输入出险情况→系统提供理赔步骤和注意事项

**内容包括**：
- 理赔步骤（拍照→报案→提交材料→审核→打款）
- 材料清单（发票、诊断证明、身份证等）
- 注意事项（时效、保留原件等）
- 常见拒赔原因
- 理赔加速技巧

## 6. 变现模型

### 6.1 收入来源

#### 来源1：CPS佣金（核心，占比80%）

**合作模式**：
用户点击产品→跳转到合作平台（水滴保/小雨伞）→购买成功→我们获得佣金

**佣金率**：
| 险种 | 佣金率 |
|------|--------|
| 重疾险 | 15-25% |
| 医疗险 | 10-20% |
| 意外险 | 20-30% |
| 年金险 | 5-10% |

**举例**：
- 用户通过我们购买重疾险，保费5000元/年
- 佣金：5000 × 20% = 1000元
- 续费佣金：每年1000元（持续收入）

---

#### 来源2：会员订阅（辅助，占比20%）

| 版本 | 价格 | 功能 |
|------|------|------|

| 月度会员 | 19.9元/月 | 无限分析 + 保单OCR + 到期提醒 |
| 年度会员 | 199元/年 | 会员功能 + 专家咨询（每月1次）+ 家人共享 |

## 8. 风险与应对

### 8.1 业务风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 需求验证失败 | 高 | 严格按需求验证阶段执行，不跳过 |
| 竞品跟进 | 中 | 保持产品迭代，持续优化 |
| 用户增长慢 | 中 | 优化获客策略，增加渠道 |
| 佣金结算问题 | 低 | 与多家平台合作，分散风险 |

### 8.2 合规风险

| 风险 | 应对措施 |
|------|----------|
| 无牌照销售保险 | 只做工具，不做销售；跳转到持牌平台 |
| 误导用户 | 免责声明醒目展示，"本工具不构成保险建议" |
| 用户数据泄露 | 数据加密存储，定期安全审计 |

### 8.3 技术风险

| 风险 | 应对措施 |
|------|----------|
| OCR识别准确率低 | 人工复核+用户反馈优化 |
| 小程序审核不通过 | 提前了解审核规则，预留修改时间 |
| 服务器故障 | 云开发自带容灾，定期备份 |

- 标题："3分钟算清你的家庭保障缺口"
- 内容：产品价值、功能介绍、定价
- 按钮："加入等待列表"

**Day 4-5：发布软文**
- 小红书：发布2-3篇"新手爸爸/妈妈保险配置"相关笔记
- 知乎：回答"保险怎么买"相关问题
- 朋友圈：分享产品理念，邀请朋友加入等待列表

**Day 6-7：收集反馈**
- 统计落地页访问量、等待列表注册数
- 分析用户评论/私信，了解真实需求

**成功标准**：
- 落地页访问量 ＞500
- 等待列表注册率 ＞8%
- 付费预购 ＞10人

### A. 常用术语解释

| 术语 | 解释 |
|------|------|
| 重疾险 | 诊断出重大疾病后，一次性赔付保额 |
| 医疗险 | 看病后凭发票报销，实报实销 |
| 寿险 | 身故后赔付保额给家人 |
| 意外险 | 意外伤害导致的身故/伤残赔付 |
| 保额 | 出险后能赔多少钱 |
| 保费 | 每年/每月要交多少钱 |
| 免责条款 | 保险公司不赔的情况 |
